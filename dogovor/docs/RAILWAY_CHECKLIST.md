# Чеклист: Railway — чтобы договор создавался и не было таймаута

На Railway SMTP заблокирован. Если не настроить почту через Resend, бэкенд пытается подключиться к SMTP, долго ждёт и ответ приходит слишком поздно — на телефоне срабатывает таймаут и видна ошибка «Запрос занял слишком много времени», хотя договор в Supabase уже создан.

## Обязательно в Railway (Variables бэкенда)

1. **RESEND_API_KEY** — ключ из [resend.com](https://resend.com) → API Keys (начинается с `re_`).
2. **RESEND_FROM_EMAIL** — для теста: `onboarding@resend.dev`; для продакшена — верифицированный домен (см. `docs/EMAIL_SMTP.md`).

После добавления переменных — **Redeploy** бэкенда (Deploy → Redeploy или новый коммит в репозиторий).

## Проверка

- В логах при генерации договора не должно быть `ERROR ... Failed to send contract email ... Network is unreachable`.
- Должна быть строка вида: `Contract PDF sent via Resend to ...`.
- На телефоне после «Сгенерировать договор» должен появляться экран успеха с номером договора, без таймаута.

## PDF с кириллицей

В образе должен быть шрифт DejaVu (в Dockerfile установлен `fonts-dejavu-core` и копируется `DejaVuSans.ttf` в `/app/fonts/`). Шрифт регистрируется в ReportLab перед рендером PDF. Если в скачанном PDF по-прежнему квадраты вместо букв — убедитесь, что задеплоена последняя версия кода (с регистрацией шрифта в `pdf_render.py`).
