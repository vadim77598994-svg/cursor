## Ваши текущие файлы (6 шт.)

Файлы лежат в `assets/` с именами вида `_______-1b6619db-....png`. Переименуйте их в:

- `Карташева.png`
- `Величко.png`
- `Еремина.png`
- `Ефремова.png`
- `Ивченко.png`
- `Никитина.png`

и скопируйте в папку `dogovor/signatures_to_upload/`.

## Загрузка в Supabase

1. Убедитесь, что в Supabase создан bucket **contracts** (тот же, что для PDF) и он публичный, либо для папки `signatures/` разрешён публичный доступ.

2. Из корня репозитория:
   ```bash
   cd dogovor/backend
   .venv/bin/python -m app.services.signature_upload ../signatures_to_upload
   ```

3. Скрипт загрузит файлы в `contracts/signatures/` и обновит поле `signature_image_url` в таблице `dogovor_staff`.

## Обновлённый шаблон договора (.docx)

Файл `ПАЙ_ОПТИКС_МЕД_ДОГ.docx` в бинарном формате, я его напрямую не читаю. Чтобы перенести изменения в наш HTML-шаблон `backend/templates/contract.html`:

- экспортируйте нужные страницы/блоки из .docx в текст или HTML, или  
- пришлите список изменений (какие переменные на каких страницах, куда вставить подпись клиента),  
и я обновлю `contract.html` и логику подставки подписей.

## Итог по подписям

- **Подписи оптометристов:** 6 файлов есть → переименовать по фамилиям → загрузить через скрипт → в договоре уже используется `staff_signature_url`.
- **Подпись клиента:** после ваших скриншотов с местами добавлю переменную (например `patient_signature_data_url`) и вставлю её в шаблон во все указанные места.
