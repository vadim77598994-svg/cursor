# Голосовой ввод для поля «Адрес прописки» — анализ

## Идея

На шаге прописки добавить опцию: **записать голос** (диктовка адреса) → распознавание речи → подстановка текста в поле «Адрес прописки».

---

## 1. Технологии

### Web Speech API (браузер, бесплатно)

- **SpeechRecognition** с `lang = "ru-RU"`.
- **Поддержка:**
  - **Chrome / Edge (десктоп и Android)** — нормальная поддержка, распознавание через серверы Google, нужен интернет.
  - **Safari (в т.ч. iOS 14.5+)** — API есть, но на практике: возможны ошибки `service-not-allowed`, на iOS 17+ часто глючит (микрофон есть, текст не появляется или неверный), после первой фразы распознавание может «зависнуть» без вызова `onend`.
  - **Firefox** — по умолчанию отключено.
- **Итог:** на планшетах в клиниках (часто iPad/Safari) стабильность **низкая**. На Android/Chrome — приемлемо для эксперимента.

### Облачные API (Yandex SpeechKit, Sber SaluteSpeech, Google Cloud Speech)

- Точность по русскому языку выше, есть настройки под длинные фразы и пунктуацию.
- Минусы: нужна регистрация, ключи, оплата (или лимиты), аудио уходит на сервер — для мед/персональных данных возможны ограничения по 152‑ФЗ и политике клиники.
- Имеет смысл рассматривать только если браузерного API категорически не хватает.

---

## 2. Точность под задачу «адрес»

- **Адрес** — смесь: названия (город, улица), сокращения (ул., д., корп., кв.), цифры (номер дома, корпуса, квартиры).
- Цифры и короткие слова («два», «корпус один») часто распознаются хуже; в навигации рекомендуют говорить явно («номер дома один»).
- Web Speech API не заточен под адреса: нет словаря улиц, нет постобработки «как адрес» — возможны опечатки, разбиение/склейка слов, неверные цифры.
- Оценка: **для голоса через браузер стабильность средняя**, ошибки 10–30% по словам (WER) для русской речи в реальных условиях — нормальная картина. Пользователь должен **обязательно проверять и править** результат.

---

## 3. Стабильность и UX

| Фактор | Оценка |
|--------|--------|
| Chrome/Edge (десктоп, Android) | Работает, при наличии интернета. |
| Safari / iPad | Нестабильно: отказ доступа, «молчание» после первой фразы, баги iOS 17. |
| Шум в клинике | Снижает точность. |
| Диктовка адреса целиком | Длинная фраза — больше ошибок; лучше короткие фразы или разбивка. |
| Без проверки пользователем | Нельзя полагаться на голос «как единственный источник» для юридического адреса. |

**Вывод:** вариант **вполне можно сделать как дополнительный, удобный способ ввода**, но только с явной подстановкой в поле и возможностью сразу отредактировать. Как единственный или основной способ для прописки — **нестабильно**, особенно на iPad.

---

## 4. Рекомендация

- **Делать голос как альтернативу** на шаге прописки: кнопка «Записать голосом» → запись → распознавание (Web Speech API, `ru-RU`) → вставка текста в поле «Адрес прописки» + **подсказка** «Проверьте и при необходимости отредактируйте».
- **Не подменять** ручной ввод и не считать голос единственным источником правды: всегда показывать результат в поле и давать его править перед переходом дальше.
- **Ограничения по платформам:** в UI можно показывать кнопку только при поддержке `SpeechRecognition` (например, не показывать в Firefox); на Safari/iOS предупреждать, что голосовой ввод может работать нестабильно, и предлагать ввод с клавиатуры.
- **Без облачных API на первом этапе:** Web Speech API достаточно для проверки спроса и сценария; при необходимости позже можно добавить опциональный бэкенд с Yandex/Sber для улучшения точности (с учётом политики по ПДн).

## 5. Краткий итог

| Вопрос | Ответ |
|--------|--------|
| Насколько стабильно? | На Chrome/Android — приемлемо для вспомогательного ввода; на Safari/iPad — нестабильно. |
| Насколько хорошо распознаёт? | По русскому и адресам — средне: возможны ошибки в цифрах и сокращениях; нужна проверка пользователем. |
| Стоит ли добавлять? | Да, как **дополнительную** опцию с обязательной подстановкой в поле и возможностью редактирования. |

Если нужно, могу расписать конкретные шаги реализации в коде (где в StepScan добавить кнопку, как вызывать SpeechRecognition и подставлять текст в поле адреса).
