# Сканирование паспорта: кто отвечает и как улучшить

## Что сейчас отвечает за сканирование

| Компонент | Файл | Роль |
|-----------|------|------|
| **UI шага «Сканирование»** | `frontend/src/components/StepScan.tsx` | Кнопка «Выбрать фото / камера», превью, кнопки «Распознать» и «Ввести вручную». По клику «Распознать» вызывается Tesseract.js. |
| **OCR** | `tesseract.js` (npm) | Работает **в браузере**. Язык сейчас только `rus`. Один снимок (разворот или что угодно). Без предобработки картинки. |
| **Парсер текста → поля** | `frontend/src/lib/parsePassportOcr.ts` | Принимает сырой текст от Tesseract, ищет по регуляркам и эвристикам и возвращает объект с полями пациента. |

**Соответствие «переменных» (поля пациента ↔ шаблон договора):**

| Поле в коде | Переменная в шаблоне | Откуда берётся сейчас |
|-------------|----------------------|------------------------|
| `patient_fio` | `{{ patient_fio }}` | Первая строка из 2–5 слов кириллицей (без цифр) — часто ошибочно («Российская Федерация» и т.п.) |
| `patient_birth_date` | — | Вторая дата ДД.ММ.ГГГГ в тексте или первая |
| `passport_series` | `{{ passport_series }}` | Первое вхождение «XX XX» (2+2 цифры) в тексте |
| `passport_number` | `{{ passport_number }}` | Первые 6 цифр подряд |
| `passport_date` | `{{ passport_date }}` | Первая дата ДД.ММ.ГГГГ |
| `passport_issued_by` | `{{ passport_issued_by }}` | Строка после «кем выдан» / «выдан» |
| `reg_address` | `{{ reg_address }}` | Текст после «место жительства» / «адрес регистрации» — с одного фото часто путается с другими блоками |

Проблемы: один кадр смешивает разворот и прописку, парсер берёт «первое подходящее», заголовки («РОССИЯ», «Паспорт») попадают в ФИО, нет предобработки изображения — из‑за этого «рандомные» значения и нестабильная работа.

## Как улучшить (без сторонних сервисов)

### 1. Два шага съёмки (разворот + прописка)

- **Шаг 1 — «Разворот с фото» (стр. 2–3):** одно фото → парсить только: ФИО, дата рождения, серия, номер, кем выдан, дата выдачи. Адрес с этого кадра не брать.
- **Шаг 2 — «Прописка»:** второе фото → парсить только адрес регистрации. Если не вышло или прописка от руки — кнопка «Ввести адрес вручную».

Так парсер не смешивает блоки и стабильнее подставляет значения по переменным.

### 2. Жёсткий парсер под структуру паспорта РФ

- **ФИО:** игнорировать строки-заголовки: «РОССИЯ», «Российская Федерация», «Паспорт», отдельные слова «Фамилия»/«Имя»/«Отчество». Брать строку из 3–4 слов кириллицей в типичной позиции (например, после метки «Фамилия» или 4–6-я осмысленная строка).
- **Серия/номер:** серия — паттерн «XX XX» (4 цифры с пробелом), номер — ровно 6 цифр подряд; при возможности привязываться к подписям «Серия»/«Номер» на странице.
- **Даты:** только ДД.ММ.ГГГГ, отбрасывать нереалистичные (будущие, до 1900). Дата рождения — обычно раньше даты выдачи.
- **Кем выдан:** строка сразу после «Кем выдан» / «выдан», обрезать по следующей дате или концу блока.
- **Адрес:** только из второго фото (блок прописки); при неудаче — не подставлять мусор, оставить пусто и предложить ввод вручную.

Парсер можно вынести в отдельные функции под каждое поле (по аналогии с переменными), чтобы было проще тестировать и править правила.

### 3. Предобработка изображения перед Tesseract

Перед вызовом `worker.recognize(image)` обрабатывать картинку в canvas или через библиотеку:

- Ресайз по длинной стороне до ~1500–2000 px (ускоряет и часто улучшает распознавание).
- Повышение контраста (усиление разницы фон/текст).
- По желанию: приведение к ч/б (бинаризация) для печатного текста.

Это сильно повышает стабильность Tesseract на фото с телефона/планшета.

### 4. Язык Tesseract

Сейчас: `createWorker("rus", 1, ...)`. В паспорте бывают латинские буквы (серия). Лучше: `rus+eng` (например загрузка двух языков в воркере), чтобы меньше ошибок в серии/номере.

### 5. Обязательная проверка пользователем (гибрид)

После OCR всегда показывать шаг «Проверка данных» с редактируемыми полями; кнопка «Далее»/«Подпись» только после перехода. Плюс на шаге сканирования оставить «Ввести вручную». Так даже при сбоях OCR пользователь может поправить значения — стабильная работа за счёт верификации.

## Сторонние сервисы (кратко)

| Вариант | Описание | Плюсы | Минусы |
|--------|----------|--------|--------|
| **Серверный Tesseract (свой бэкенд)** | Фронт шлёт фото на ваш API; на сервере PIL/OpenCV + pytesseract, парсер, ответ JSON с полями. | Полный контроль, одна предобработка и один парсер, стабильное окружение. | Нужен Tesseract в Docker (Railway и т.д.), фото паспорта уходят на ваш сервер (политика хранения/безопасность). |
| **Google Cloud Vision / Document AI** | Отправка изображения в API, получение текста или структуры. | Высокое качество печатного текста. | Платно, данные уходят в Google; для паспортов — вопросы по 152-ФЗ и политике клиники. |
| **ABBYY Cloud OCR** | Облачный OCR под документы. | Хорошее качество документов. | Платно, данные в ABBYY. |
| **Яндекс Vision** | Аналогично — облачный распознавание. | Удобно в экосистеме Яндекса. | Данные покидают ваш контур, стоимость. |

Для стабильной работы без передачи паспорта третьим лицам оптимально сначала улучшить текущий стек (два фото + парсер по переменным + предобработка + rus+eng + проверка пользователем). Серверный Tesseract — следующий шаг, если нужна ещё большая стабильность и допустима передача фото на ваш бэкенд. Облачные OCR — только при проработанной политике ПДн и согласии.

---

## Рекомендация для стабильной работы

1. **Сделать сейчас (в текущем стеке):**  
   - Два шага съёмки: разворот → поля кроме адреса; прописка → только адрес (с кнопкой «Ввести адрес вручную»).  
   - Ужесточить парсер под структуру паспорта РФ и явный маппинг на переменные (`patient_fio`, `passport_series`, и т.д.).  
   - Предобработка изображения перед Tesseract (ресайз, контраст).  
   - Язык `rus+eng`.  
   - Обязательный шаг проверки и ручная правка (уже есть, важно не пропускать его).

2. **Если качества всё ещё не хватает:**  
   Вынести OCR на бэкенд: `POST /api/v1/passport/recognize`, изображение → предобработка → Tesseract → тот же (или усиленный) парсер → JSON с полями. Фронт только отправляет фото и подставляет ответ в форму.

3. **Сторонние облачные OCR** — рассматривать только при готовности передавать паспорт в сторонний сервис и при проработанной юридической/безопасностной позиции.
