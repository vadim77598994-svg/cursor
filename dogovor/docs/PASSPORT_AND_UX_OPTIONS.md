# Варианты: сканирование паспорта, два фото, подпись на телефоне, подвисание

Краткий разбор трёх проблем и варианты решений.

---

## 1. Подвисание при «Сгенерировать договор»

**Причина:** Запрос к бэкенду (генерация PDF) может занимать 5–30 секунд; таймаута нет, при долгом ответе или обрыве кажется, что «зависло».

**Что сделано:** Добавлен таймаут запроса (60 с), явный текст «Создание договора…» и сообщение при ошибке/таймауте.

**При необходимости:** Увеличить таймаут в `api.ts` или показывать прогресс «Формируем PDF…» с анимацией.

---

## 2. Подпись не реагирует на палец (iPhone)

**Причина:** У canvas обрабатываются только события мыши (`mouse*`). На сенсорных устройствах срабатывают события `touch*`; без их обработки рисование пальцем не работает.

**Что сделано:** Добавлены обработчики `onTouchStart`, `onTouchMove`, `onTouchEnd`: перевод координат touch в координаты canvas, `preventDefault()` чтобы страница не скроллилась. Подпись пальцем должна работать на телефоне и планшете.

---

## 3. Сканирование паспорта: качество и два фото

### Текущее состояние

- Одно фото (разворот или прописка).
- Tesseract.js только с языком `rus`, без предобработки изображения.
- Парсер ищет первое подходящее по эвристикам (даты, ФИО, адрес), из‑за чего в поля могут попадать мусор или не та строка (например, «Российская Федерация» вместо ФИО).

### Варианты по приоритету

#### A. Два шага съёмки + доработка OCR (рекомендуется)

- **Шаг 1 — «Разворот с ФИО и номером»:** одно фото (страницы 2–3). По нему заполняем: ФИО, дата рождения, серия/номер, кем выдан, дата выдачи.
- **Шаг 2 — «Прописка»:** второе фото (страница с адресом). По нему заполняем только адрес регистрации.

**Плюсы:** Чёткое разделение: с одного кадра не тянем и ФИО, и прописку; меньше путаницы в парсере.  
**Дополнительно:** предобработка изображения (ресайз до ~1500–2000 px по ширине, контраст, при желании ч/б) перед отправкой в Tesseract; язык `rus+eng`; парсер ФИО — пропускать строки-заголовки («РОССИЯ», «Паспорт», «Фамилия» отдельным словом), брать строку из 3–4 слов кириллицей.  
**Прописка от руки:** Tesseract с рукописным текстом работает плохо. Варианты: (1) для второго фото только автоматическое распознавание, при неудаче — кнопка «Ввести адрес вручную»; (2) не обещать распознавание рукописного, явно написать «Если прописка от руки — введите адрес вручную».

**Оценка:** Реализуемо в текущем стеке, заметно улучшает точность без облачных API.

#### B. Внешний OCR (Google Vision, ABBYY и т.п.)

- Отправка фото на API, получение готового текста или структуры полей.

**Плюсы:** Часто выше качество печатного текста.  
**Минусы:** Стоимость, зависимость от сервиса, передача персональных данных на сторонний сервер (важно для паспортов).  
**Оценка:** Имеет смысл только при жёстких требованиях к точности и готовности платить и выносить данные.

#### C. Только ручной ввод с подсказками

- Убрать авто-распознавание, оставить форму с полями и короткими подсказками («Серия: 4 цифры», «Номер: 6 цифр» и т.д.).

**Плюсы:** Нет ошибок OCR, быстрая реализация.  
**Минусы:** Медленнее для оператора, больше ручной работы.  
**Оценка:** Резервный вариант или дополнение к «попробовать OCR → не вышло → ввести вручную».

### Рекомендация по шагам

1. **Сначала:** Включить **два фото** (разворот + прописка) и разделить парсинг: с первого — ФИО, даты, серия/номер, выдан; со второго — только адрес. Улучшить парсер (исключить заголовки, строже правила для ФИО) и предобработку изображения перед Tesseract.
2. **Прописка от руки:** Не полагаться на OCR. После фото прописки показывать поле «Адрес регистрации» с подсказкой: «Если не распозналось или написано от руки — введите вручную». При желании можно позже пробовать облачный handwriting API только для этого поля (с учётом бюджета и персональных данных).
