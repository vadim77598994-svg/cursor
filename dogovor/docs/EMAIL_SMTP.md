# Отправка договора на email (SMTP)

После генерации договора PDF можно отправить клиенту на указанный email.

## Настройка

В `backend/.env` заданы переменные (уже заполнены для support@pyeoptics.com):

- `SMTP_HOST` — хост (например `smtp.yandex.ru`)
- `SMTP_PORT` — порт (465 для SSL)
- `SMTP_USER` — логин
- `SMTP_PASSWORD` — пароль (или пароль приложения)
- `SMTP_FROM_EMAIL` — адрес отправителя
- `SMTP_FROM_NAME` — имя отправителя («Пай Оптикс»)

Файл `.env` в git не коммитится — не публикуйте пароль в репозитории.

## Поведение

1. На шаге «Проверка данных» можно указать **Email для отправки договора** (поле необязательное).
2. После подписания и нажатия «Сгенерировать договор» бэкенд создаёт PDF, сохраняет в Storage и, если email указан и SMTP настроен, отправляет письмо с вложением PDF на этот адрес.
3. В ответе API приходит `email_sent: true/false`; на экране успеха отображается «Договор отправлен на email клиента», если письмо отправлено.

Если SMTP не настроен или отправка не удалась, договор всё равно создаётся и сохраняется; клиент просто не получит письмо.

## Проверка подключения к почте

На проде открой в браузере (подставь URL бэкенда):

```
https://ТВОЙ-БЭКЕНД-URL/debug/smtp-check
```

В ответе будет:
- `smtp_configured` — заданы ли SMTP_HOST, SMTP_USER, SMTP_PASSWORD
- `connection_ok` — удалось ли подключиться к серверу и авторизоваться
- `error` — текст ошибки, если подключение не удалось (пароль в ответ не попадает)

По `error` можно понять причину: неверный пароль приложения, блокировка порта, неверный хост и т.д.

## Яндекс: пароль приложения

Для Яндекса нужен **пароль приложения**, а не пароль от ящика. Создать: [ID Яндекса](https://id.yandex.ru) → Безопасность → Пароли приложений. В Railway в `SMTP_PASSWORD` подставь именно этот пароль.
