# OCR: как сделать шаг «Сканирование паспорта»

Сейчас на шаге 2 пользователь нажимает «Ввести данные вручную» и переходит к форме. Чтобы **автоматически подставлять** ФИО, серию/номер паспорта, кем выдан, дату выдачи и адрес регистрации из фото паспорта, нужно добавить распознавание текста (OCR).

---

## Что такое OCR в этом проекте

**OCR** (Optical Character Recognition) — это распознавание текста на изображении. В нашем случае: камера планшета/телефона снимает разворот паспорта (страницы 2–3), картинка отправляется в движок распознавания, он возвращает текст, мы **парсим** этот текст и подставляем поля в форму (ФИО, серия, номер, кем выдан, дата, адрес регистрации).

---

## Варианты реализации

### Вариант 1: Tesseract.js (в браузере, бесплатно)

**Суть:** библиотека Tesseract.js работает прямо в браузере. Пользователь делает снимок (или выбирает файл), мы передаём изображение в Tesseract, получаем строку текста и разбираем её по полям.

**Плюсы:** не нужен сервер для OCR, бесплатно, быстро внедрить.  
**Минусы:** качество хуже, чем у платных/серверных решений; паспорт нужно снимать чётко, хорошо освещённо; парсинг полей по правилам (серия/номер, «кем выдан» и т.д.) нужно написать самим.

**Что сделать по шагам:**

1. **Установить пакет в фронте**
   ```bash
   cd dogovor/frontend
   npm install tesseract.js
   ```

2. **Добавить на шаг «Сканирование» (Step 2):**
   - Поле выбора файла (input type="file" accept="image/*") или доступ к камере (navigator.mediaDevices.getUserMedia), чтобы пользователь сделал фото паспорта.
   - Кнопку «Распознать»: по клику брать выбранное/снятое изображение, вызывать Tesseract.recognize(image, 'rus'), получать text.
   - Парсер: по тексту найти ФИО (обычно первые строки на 2-й странице), серию/номер (две группы цифр), «кем выдан», дату выдачи, адрес регистрации — и заполнить state формы (те же поля, что сейчас в StepReview).
   - После распознавания автоматически переходить на шаг 3 (проверка данных) с уже заполненными полями; пользователь при необходимости правит и жмёт «Далее — подпись».

3. **Безопасность (как в переписке):** после того как текст извлечён и отправлен на бэкенд (или хотя бы через 30 секунд), удалить объект изображения из памяти (обнулить blob URL, не хранить фото на устройстве).

4. **Парсинг полей:** правила зависят от формата вывода Tesseract. Обычно:
   - ФИО — строки после «Фамилия», «Имя», «Отчество» или одна строка в начале блока.
   - Серия и номер — два числа (4 цифры и 6 цифр), часто с пробелом.
   - «Кем выдан» — строка после подписи «кем выдан» или после даты выдачи.
   - Дата выдачи — искать паттерн ДД.ММ.ГГГГ.
   - Адрес регистрации — блок после «Место жительства» или «Адрес регистрации».

Можно задать простые регулярные выражения или пошаговый разбор по строкам и подстрокам.

### Вариант 2: Regula Document Reader (платный SDK)

**Суть:** Regula предоставляет SDK (в т.ч. для веба), который сам находит документ на фото, выравнивает, распознаёт и возвращает **структурированные поля** (фамилия, имя, отчество, серия, номер, дата выдачи, орган выдачи, адрес и т.д.). Не нужно писать парсер.

**Плюсы:** высокая точность, меньше своей логики.  
**Минусы:** платная лицензия, нужна регистрация и ключ API / интеграция SDK.

**Что сделать по шагам:**

1. Зарегистрироваться на сайте Regula, получить ключ/доступ к Document Reader SDK для Web.
2. Подключить SDK в проект (по инструкции Regula): скрипт или npm-пакет.
3. На шаге «Сканирование»: показывать камеру через их компонент (или свой видеопоток + передавать кадр в SDK). SDK вернёт JSON с полями документа.
4. Маппинг: взять из ответа Regula поля (например surname, name, patronymic, documentNumber, issueDate, issuingAuthority, registrationAddress) и записать их в тот же state формы, что и сейчас (patient_fio, passport_series, passport_number, passport_issued_by, passport_date, reg_address). Формат даты/серии при необходимости привести к нашему (например ДД.ММ.ГГГГ).
5. После успешного распознавания перейти на шаг 3; фото не хранить (удалить через 30 сек или сразу после извлечения данных).

---

### Вариант 3: Серверный OCR (Python на бэкенде)

**Суть:** фронт отправляет изображение (файл или base64) на наш FastAPI; на сервере вызывается Tesseract (py-tesseract) или другая библиотека (например EasyOCR), результат парсится и возвращается JSON с полями. Фронт только отображает камеру/загрузку файла и подставляет ответ в форму.

**Плюсы:** можно использовать более мощный движок, единый парсер на Python.  
**Минусы:** нужна установка Tesseract/EasyOCR на сервере, нагрузка на бэкенд, передача фото по сети (нужна политика безопасности и не хранить фото дольше времени обработки).

**Что сделать по шагам:**

1. На сервере установить Tesseract: `apt-get install tesseract-ocr tesseract-ocr-rus` (или аналог для macOS/Windows).
2. В бэкенде добавить зависимость: `pytesseract` и `Pillow`. Эндпоинт, например: `POST /api/v1/passport/recognize`, body: multipart file (image) или JSON с base64.
3. В обработчике: открыть изображение, вызвать pytesseract.image_to_string(..., lang='rus'), распарсить текст (аналогично варианту 1) и вернуть JSON: `{ "patient_fio": "...", "passport_series": "...", ... }`.
4. На фронте: на шаге 2 отправить фото на этот эндпоинт, по ответу заполнить форму и перейти на шаг 3. Изображение на сервере не сохранять, удалять после ответа.

---

## Что выбрать

- **Быстро и бесплатно, MVP:** Вариант 1 (Tesseract.js в браузере).  
- **Максимальное качество и готовые поля:** Вариант 2 (Regula), если есть бюджет.  
- **Контроль на своей стороне, без браузерных ограничений:** Вариант 3 (серверный OCR).

Можно начать с Tesseract.js: один компонент «Сканирование» с выбором файла/камерой, вызов recognize, простой парсер по строкам и регуляркам, заполнение формы и переход на шаг 3. Потом при необходимости заменить на Regula или серверный OCR, не меняя остальной поток (форма и подпись остаются те же).

## Ограничения и требования (учесть при реализации Tesseract.js)

**Устройство и браузер:** основное использование — **iPad**, браузер Safari. Всё должно работать в веб-приложении (тот же Next.js), без отдельного нативного приложения. При желании страницу можно добавить на домашний экран (Add to Home Screen).

**Камера и фото:**
- Доступ к камере через `navigator.mediaDevices.getUserMedia`; при первом использовании Safari запросит разрешение.
- Обязательно дать возможность **выбрать фото из галереи** (input file или picker), а не только снять с камеры — так проще переснять при плохом качестве.
- После распознавания **не хранить фото**: сбросить blob URL и ссылки на изображение не позднее чем через ~30 секунд (по требованиям проекта).

**Производительность и UX:**
- Первая загрузка страницы с Tesseract.js требует сети (подгрузка движка/worker). Дальше распознавание может работать офлайн.
- На iPad распознавание одного кадра может занимать **5–15 секунд**. Показывать индикатор «Распознаём…» и блокировать повторное нажатие до завершения.
- Подсказка на экране: «Разместите паспорт в кадре, избегайте бликов» — для лучшего качества OCR.

**Парсинг:** вывод Tesseract по паспорту может «плыть» (переносы, лишние пробелы). Заложить запас по разбору: несколько вариантов регулярных выражений, поиск по ключевым словам («кем выдан», «дата выдачи», «место жительства»), проверка формата даты ДД.ММ.ГГГГ.
