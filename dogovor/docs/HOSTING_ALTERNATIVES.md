# Альтернативы хостинга бэкенда: бесплатные и платные

Краткий разбор вариантов для деплоя FastAPI-бэкенда (dogovor) и насколько они подходят для **стабильной работы** в кабинетах (планшеты, подписание договоров).

---

## Что важно для нашего сценария

- **Отклик без долгих задержек** — сотрудник и пациент у экрана; первый запрос после простоя не должен занимать 30–60 секунд (cold start).
- **Предсказуемая доступность** — сервис не «засыпает» в самый момент подписания.
- **HTTPS, переменные окружения** — для Supabase и SMTP.
- **Без своего диска** — данные в Supabase, файлы в Storage, локальный диск не нужен.

---

## Бесплатные и условно-бесплатные варианты

### 1. Render (Free)

| Параметр | Значение |
|----------|----------|
| Регистрация | Без карты |
| Часы в месяц | 750 (хватает на 24/7 один сервис) |
| Главное ограничение | **Сервис «засыпает» после 15 минут без запросов.** Пробуждение ~30–60 сек. |
| RAM / CPU | 512 MB, 0.1 CPU |
| Свой домен, SSL | Да |

**Подойдёт ли для стабильной работы:** частично. Если в кабинете несколько минут никто не нажимает «Сгенерировать договор», следующий запрос может упереться в холодный старт. Для теста и малой нагрузки — ок; для постоянной работы с пациентами — риск задержек в момент подписания.

---

### 2. Fly.io

| Параметр | Значение |
|----------|----------|
| Регистрация | Карта для верификации (списываний может не быть в пределах free allowance) |
| Free allowance | Ограниченный объём ресурсов в месяц (vCPU, RAM, трафик). Точные лимиты — на fly.io/docs. |
| «Засыпание» | Зависит от конфига; можно сделать машинку всегда включённой в рамках лимитов. |
| Сложность | Выше: нужен Docker/образ или fly.toml. |

**Подойдёт ли:** при готовности настроить деплой и уложиться в free allowance — да, можно добиться более стабильного отклика, чем у Render free. Для «включил и забыл» — сложнее, чем Railway/Render.

---

### 3. PythonAnywhere (Free)

| Параметр | Значение |
|----------|----------|
| Регистрация | Без карты |
| Ограничения | 1 веб-приложение, **100 CPU-секунд в день**, 512 MB диска. У новых аккаунтов веб-приложение может иметь ограниченный срок (например, 1 месяц). |
| «Засыпание» | Нет явного sleep, но лимит CPU очень мал. |

**Подойдёт ли:** нет. 100 CPU-секунд в день — слишком мало для стабильного API при любом количестве обращений. Подходит только для экспериментов.

---

### 4. Koyeb (Free tier)

| Параметр | Значение |
|----------|----------|
| Регистрация | Обычно просят карту для верификации (холд $1 и т.п.) |
| Особенность | Scale-to-zero: при отсутствии трафика инстанс останавливается, при запросе — холодный старт. |
| Ресурсы | Ограниченный бесплатный объём (vCPU/RAM/часы). |

**Подойдёт ли:** как и Render free — возможны задержки при первом запросе после простоя. Для «стабильно в кабинете» — не идеально.

---

### 5. Railway

| Параметр | Значение |
|----------|----------|
| Регистрация | Пробный период с кредитами, далее Free (~$1/мес) или платные планы |
| Платные планы | От ~$5/мес (Hobby) — сервис всегда включён, без принудительного «сна». |
| Плюсы | Простое подключение GitHub, переменные, домен, предсказуемая работа на платном плане. |

**Подойдёт ли:** бесплатный уровень — с теми же рисками по лимитам; для стабильной работы в проде разумно рассматривать платный план.

---

## Сводка: стабильная работа

| Вариант | Бесплатно | Стабильно без холодных стартов | Сложность настройки |
|--------|-----------|--------------------------------|----------------------|
| **Render Free** | Да (750 ч/мес) | Нет (сон 15 мин, долгое пробуждение) | Низкая |
| **Fly.io** | В пределах free allowance | Реализуемо (если сервис не гонять в 0) | Средняя/высокая |
| **PythonAnywhere Free** | Да | Нет (жёсткие лимиты CPU) | Низкая |
| **Koyeb Free** | Условно (часто с картой) | Нет (scale-to-zero) | Средняя |
| **Railway Free** | Очень ограниченно | Сомнительно (лимиты кредитов) | Низкая |
| **Railway Hobby (~$5/мес)** | Нет | Да | Низкая |
| **Render Paid** | Нет | Да (без принудительного сна) | Низкая |

**Вывод:**  
- **Бесплатные варианты** (Render, Koyeb, Fly.io free, Railway free) либо «усыпляют» сервис, либо сильно ограничивают ресурсы. Для сценария «пациент подписывает здесь и сейчас» это даёт риск задержек или отказов в момент подписания.  
- **Для стабильной работы** разумно считать минимальным вариантом **платный тариф** (~$5/мес): Railway Hobby, Render Starter или аналог. Тогда сервис всегда включён, холодных стартов нет, поведение предсказуемо.

---

## Рекомендация по шагам

1. **Проверить идею на бесплатно:** задеплоить на **Render Free** (или Railway free), пройти сценарий договора, оценить, насколько мешает задержка после простоя.
2. **Если задержки критичны:** перейти на **платный план** (Railway Hobby или Render) — те же настройки (Root Directory, переменные, команда запуска), меняется только тариф.
3. **Если готовы возиться с конфигом:** рассмотреть **Fly.io** в пределах free allowance и без scale-to-zero — можно получить «всегда включён» бесплатно, но настройка сложнее.

Актуальные лимиты и цены всегда уточняйте на сайтах: [render.com](https://render.com), [fly.io](https://fly.io), [railway.app](https://railway.app), [koyeb.com](https://koyeb.com).
