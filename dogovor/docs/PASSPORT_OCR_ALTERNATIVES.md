# Распознавание паспорта: бюджетные аналоги и самостоятельная реализация

## Рынок платных API (кратко)

| Сервис | Что умеет | Цена | Комментарий |
|--------|-----------|------|--------------|
| **Beorg (Биорг.Паспорт)** | Разворот 2–3 + прописка в одном запросе (PASSPORT_REG), ФИО, серия, номер, адрес | От **4 ₽/распознавание** (пакет от 10 000 ₽ = 2500 шт.). Подписка от ~5000 ₽/мес | Дорого при малом объёме; выгодно при больших объёмах. 25 распознаваний бесплатно при регистрации. |
| **Regula Document Reader** | Паспорта, документы, MRZ | **По запросу** (нет открытых тарифов), trial 30 дней | Корпоративный сегмент, обычно дороже Beorg. |
| **Google Cloud Vision / Document AI** | Общий OCR, Document AI — парсеры под документы | Vision: **$1.50 / 1000** запросов (первые 1000/мес бесплатно). Document AI: **$0.10 / документ** под парсеры | Нет готового парсера именно под паспорт РФ с пропиской; пришлось бы самим маппить поля. |
| **DaData** | **Нет OCR.** Только проверка серии/номера по МВД, стандартизация, «кем выдан» | Стандартизация ~20 коп/паспорт; подписка от 14 000 ₽/год | Использовать только для проверки уже введённых данных, не для распознавания с фото. |

**Итог по платным:** при необходимости и разворота, и прописки самый прозрачный по цене — Beorg (от 4 ₽/вызов), но входной порог 10 000 ₽. Более бюджетной «готовой» альтернативы под паспорт РФ с пропиской на рынке почти нет.

## Самостоятельная реализация (Tesseract + предобработка)

### Почему Tesseract «ломается» на серии и номере

На развороте паспорта РФ серия и номер напечатаны **вертикально** (столбиком). Tesseract обучен в основном на горизонтальный текст, поэтому вертикальные цифры распознаёт плохо или выдаёт мусор.

### Идея: вырезать область и повернуть

1. **Фиксированная область на развороте.** На типичном фото разворота 2–3 стр. блок с серией/номером обычно находится в **левой части** кадра (страница с фото), в средней/нижней зоне по высоте.
2. **Crop по относительным координатам.** Вырезать прямоугольник, например: по ширине 5–35% от левого края, по высоте 35–85%. Точные проценты можно подобрать по тестовым фото (зависит от кадрирования).
3. **Поворот на 90°.** Повернуть вырезанный фрагмент на 90° по часовой стрелке. Тогда вертикальный столбик цифр станет горизонтальной строкой.
4. **Отдельный прогон Tesseract** только по этому фрагменту (режим «одна строка» или «один блок»), затем парсить из текста 4 цифры (серия) и 6 цифр (номер).

Такой подход используют открытые проекты (paSport_eye, paspread и др.): OpenCV + Tesseract, предобработка и часто отдельная обработка «сложных» зон.

### Что можно сделать у нас (без нового бэкенда)

- **Фронт (браузер):** на том же фото разворота:
  - рисовать на canvas область (например, левая часть по заданным процентам);
  - вырезать её, повернуть на 90°, отдать в Tesseract.js;
  - из результата вытащить только цифры и по правилам (4 цифры = серия, 6 цифр = номер) заполнить поля.
- **Плюсы:** бесплатно, без внешнего API, фото никуда не уходит.
- **Минусы:** нужно подобрать зону под ваши типовые ракурсы; при сильном наклоне/смещении паспорта зона может «уехать».

### Технические данные для стабильной реализации (официальные источники)

**Источник:** Приказ МВД России от 13.11.2017 N 851 (внутренний паспорт РФ), КонсультантПлюс — структура МЧЗ.

**Расположение МЧЗ:** нижняя четверть третьей страницы разворота (2–3 стр.). По ГОСТ: верхняя строка МЧЗ — не менее 17,9 мм от нижнего края документа, длина полосы до 114 мм, шрифт OCR-B.

**Структура второй строки МЧЗ (44 символа, нумерация с 1):**

| Позиции | Элемент | Описание |
|---------|---------|----------|
| 1–9 | Серия и номер | **Три первые цифры серии** + **шесть цифр номера** (единым 9-значным числом). Пример: серия 4601, номер 123456 → запись `460123456` (позиции 1–3 = 460, 4–9 = 123456). |
| 10 | Контрольная цифра | По позициям 1–9, алгоритм «Модуль 10», вес 731. |
| 11–13 | Гражданство | RUS. |
| 14–19 | Дата рождения | YYMMDD. |
| 20 | Контрольная цифра | По дате рождения. |
| 21 | Пол | M / F. |
| 22–28 | Срок действия и контрольная цифра | Заполнители для внутреннего паспорта. |
| **29** | **4-я цифра серии** | **Последняя цифра серии паспорта** (в позициях 1–3 только первые три цифры серии). |
| 30–41 | Дата выдачи (YYMMDD) + код подразделения (6 цифр) | Доп. данные. |
| 42–44 | Заполнитель и контрольные цифры | |

**Итого для серии и номера:** серия (4 цифры) = позиции 1–3 + позиция 29; номер (6 цифр) = позиции 4–9. Текст МЧЗ — **горизонтальный**, хорошо читается Tesseract.

**Практическая реализация:** вырезать **нижнюю часть** кадра разворота (например, нижние 20–30% по высоте), распознать две строки Tesseract’ом, найти строку из 44 символов (цифры и латиница), распарсить по позициям выше. Контрольную цифру (позиция 10) можно проверять по алгоритму 731 для отсечения мусора.

### Open-source проекты для ориентира

- **paSport_eye** (OpenCV, Tesseract, Dlib) — распознавание разворота паспорта РФ, есть режим «улучшенного» распознавания трудных зон.
- **paspread** — библиотека для чтения данных с паспорта РФ (OpenCV + Tesseract).
- **Russian_passport_text_recognition** — несколько способов чтения кириллицы из паспорта (Python, Tesseract).
- **RussianDocsOCR** — распознавание УЛ РФ (паспорта, ВУ, СНИЛС и т.д.).

Во всех случаях основа — предобработка изображения и точечный запуск Tesseract по нужным областям (в т.ч. с поворотом вертикального текста).

## Рекомендация по приоритетам

1. **Сделать свой «crop + rotate» под серию/номер** на фронте (или на бэкенде на Python с OpenCV + Tesseract), оставить Beorg опционально для тех, кому нужна максимальная стабильность и прописка «из коробки».
2. **Платный API подключать** только если после настройки зоны и поворота качество всё ещё не устраивает или нужна гарантированная прописка без доработок.
3. **DaData** использовать только для **проверки** уже введённых серии и номера (формат, база недействительных МВД), не для OCR.

## Что уже сделано в проекте (DIY без платного API)

В коде реализован вариант **«crop + rotate»** для серии и номера:

1. **`preprocessImage.ts`** — функция `extractSeriesNumberRegion(file)`:
   - вырезает область разворота по относительным координатам (левая часть кадра ~5–38% по ширине, ~35–88% по высоте);
   - поворачивает её на 90° по часовой стрелке, чтобы вертикальный столбик цифр стал горизонтальной строкой;
   - возвращает `Blob` для второго прогона Tesseract.

2. **`parsePassportOcr.ts`** — функция `parseSeriesNumberFromCrop(rawText)`:
   - из текста, полученного с повёрнутой области (в основном цифры), извлекает 4 цифры (серия) и 6 цифр (номер);
   - отсекает варианты, похожие на год (1950–2030).

3. **`StepScan.tsx`** — при нажатии «Распознать разворот» (Tesseract):
   - сначала распознаётся полное изображение разворота → ФИО, даты, кем выдан;
   - затем вырезается область серии/номера, поворачивается, распознаётся вторым прогоном Tesseract;
   - серия и номер из второго прогона подставляются в результат (перезаписывают, если основное распознавание их не нашло).

Зону вырезки (проценты ширины/высоты) при необходимости можно подстроить под ваши типовые фото (в `extractSeriesNumberRegion` в `preprocessImage.ts`). Платная подписка при этом не нужна; Beorg остаётся опцией для максимальной стабильности и прописки в одном запросе.

## Реализация по МЧЗ (официальная структура)

В коде добавлен **приоритетный** путь по машиночитаемой зоне (приведённые выше технические данные):

1. **`preprocessImage.ts`** — `extractMRZRegion(file)`:
   - вырезает **нижние 28%** кадра разворота (зона МЧЗ по регламенту — нижняя четверть страницы);
   - возвращает `Blob` для Tesseract (горизонтальный текст, распознаётся стабильнее).

2. **`parsePassportOcr.ts`** — `parseSeriesNumberFromMRZ(line2)` и `parseMRZRawForSeriesNumber(rawText)`:
   - по второй строке МЧЗ (44 символа): позиции 1–3 = первые 3 цифры серии, 4–9 = номер, позиция 29 = 4-я цифра серии (Приказ МВД 851);
   - из сырого текста OCR ищется строка длиной ≥44 символа или последовательность из 9 цифр + при необходимости следующий символ для 4-й цифры серии.

3. **`StepScan.tsx`** — порядок при распознавании разворота:
   - полное изображение → Tesseract → ФИО, даты, кем выдан;
   - **сначала** вырез МЧЗ → Tesseract → парсинг по официальной структуре (серия/номер);
   - если по МЧЗ серия/номер не получены — вырез вертикальной области → поворот 90° → Tesseract → `parseSeriesNumberFromCrop`.

**Источники данных:** Приказ МВД России от 13.11.2017 N 851 (структура нижней строки МЧЗ), КонсультантПлюс — описание позиций и контрольных цифр.
