# Пошаговый план: Договор (сканирование и подписание)

## Сделано

1. **Supabase:** схема в `supabase/001_contracts_schema.sql`, bucket `contracts`, подписи в `signatures/` (ASCII-имена).
2. **Backend:** FastAPI — локации, оптометристы, генерация договора (номер, Jinja2 → PDF, Storage, БД). CORS для фронта.
3. **Frontend:** степпер — кабинет/врач → сканирование (OCR или ввод) → проверка данных (в т.ч. email) → подпись → API, экран успеха.
4. **Шаблон договора:** `contract.html` с переменными, подписи оптометриста и клиента в 3+4 местах.
5. **OCR:** Tesseract.js (rus), см. `docs/OCR_КАК_СДЕЛАТЬ.md`.
6. **Подписи оптометристов:** загрузка через `app.services.signature_upload`, URL в БД, подстановка в PDF. См. `docs/SIGNATURES_SETUP.md`.
7. **Отправка PDF на email:** SMTP (Яндекс, support@pyeoptics.com), поле «Email для отправки договора» на шаге проверки. См. `docs/EMAIL_SMTP.md`.

## Дальше

8. **Безопасность при запуске:** секреты только в env, HTTPS и CORS на проде, доступ к API. См. `docs/SECURITY_DEPLOY.md`.
9. **Домен и хостинг:** вынести фронт и бэкенд на прод (Vercel + Railway / другой хост), настроить домен, выдать переменные окружения. См. **`docs/DEPLOY.md`**.
10. **Доработки фронта:** дизайн, доработки под планшеты, при необходимости — простая авторизация для кабинетов.
11. **Опционально:** webhook при подписании для CRM; WhatsApp/Telegram для отправки договора.
