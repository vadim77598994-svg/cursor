# Пошаговый план: Договор (сканирование и подписание)

## Сделано

1. **Supabase:** схема в `supabase/001_contracts_schema.sql`, bucket `contracts`, подписи в `signatures/` (ASCII-имена).
2. **Backend:** FastAPI — локации, оптометристы, генерация договора (номер, Jinja2 → PDF, Storage, БД). CORS для фронта.
3. **Frontend:** степпер — кабинет/врач → сканирование (OCR или ввод) → проверка данных (в т.ч. email) → подпись → API, экран успеха.
4. **Шаблон договора:** `contract.html` с переменными, подписи оптометриста и клиента в 3+4 местах.
5. **OCR:** Tesseract.js (rus), см. `docs/OCR_КАК_СДЕЛАТЬ.md`.
6. **Подписи оптометристов:** загрузка через `app.services.signature_upload`, URL в БД, подстановка в PDF. См. `docs/SIGNATURES_SETUP.md`.
7. **Отправка PDF на email:** два провайдера по выбору. **Railway:** SMTP заблокирован → Resend API (`EMAIL_PROVIDER=resend`, `RESEND_API_KEY`). **Timeweb (после переезда):** SMTP Яндекса (`EMAIL_PROVIDER=smtp`, `SMTP_*`). См. `docs/EMAIL_SMTP.md`, `docs/EMAIL_STRATEGY.md`.

## Дальше

8. **Почта на Railway:** настроить Resend (ключ, `RESEND_FROM_EMAIL` в Variables), проверить отправку договора на email.
9. **Безопасность при запуске:** секреты только в env, HTTPS и CORS на проде. См. `docs/SECURITY_DEPLOY.md`.
10. **Домен и хостинг:** прод на Railway (фронт + бэкенд); позже переезд на Timeweb — тогда переключить почту на SMTP.
11. **Доработки фронта:** дизайн, планшеты; при необходимости — простая авторизация для кабинетов.
12. **Опционально:** webhook при подписании для CRM; WhatsApp/Telegram для отправки договора.

## До идеала (чеклист)

Когда будем доводить проект до идеала, напомнить себе про:

- [ ] **Почта на Railway:** Resend подключён, письма с договором доходят до клиента.
- [ ] **Сканирование паспорта:** два фото (разворот с ФИО + прописка), улучшенный OCR и парсер; при ручной прописке — ввод адреса вручную. См. `docs/PASSPORT_AND_UX_OPTIONS.md`.
- [ ] **Подпись на телефоне:** touch уже добавлен; при необходимости проверить на реальных устройствах.
- [ ] **Генерация договора:** таймаут и индикация «Подождите…» уже есть; при долгих ответах — при необходимости увеличить таймаут.
- [ ] **Переезд на Timeweb:** выставить `EMAIL_PROVIDER=smtp`, SMTP_* (Яндекс), проверить отправку и деплой.
