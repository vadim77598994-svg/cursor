# Подписи оптометристов

## Требования к PNG (чтобы подпись в договоре была чёткой)

- **Разрешение:** желательно **400–800 px по длинной стороне** (или выше). Исходник с хорошим разрешением встраивается в PDF в 8× от отображаемого размера (336 px по высоте), поэтому подпись остаётся чёткой при печати и зуме.
- **Формат:** PNG (без потери качества). Можно с прозрачным фоном (RGBA).
- **Не делать:** не используйте маленькие картинки (меньше ~150 px по высоте) — мелкие изображения мы не увеличиваем (будет размыто).

Имена файлов должны совпадать с фамилиями оптометристов (допускается «ё» в файле: `Карасёва.png`, `Карташёва.png`):

| Файл | ФИО |
|------|-----|
| `Карташева.png` (или `Карташёва.png`) | Карташева Татьяна Владиславовна |
| `Величко.png` | Величко Ирина Владимировна |
| `Еремина.png` | Еремина Анжелика Юрьевна |
| `Ефремова.png` | Ефремова Светлана Павловна |
| `Ивченко.png` | Ивченко Виолетта Евгеньевна |
| `Никитина.png` | Никитина Елена Михайловна |
| `Коваленко.png` | Коваленко Николай Валентинович (СПб) |
| `Карасева.png` (или `Карасёва.png`) | Карасева Виктория Викторовна (Москва) |

Загрузка в Supabase (из корня репозитория):
```bash
cd dogovor/backend
.venv/bin/python -m app.services.signature_upload ../signatures_to_upload
# или без аргумента (скрипт возьмёт dogovor/signatures_to_upload):
# .venv/bin/python -m app.services.signature_upload
```

## Как проверить, что подпись подставлена верно

**Случай «у Карасевой в договоре подпись Ивченко»** — значит в привязке ошибка. Сделайте два шага:

1. **Проверить, что в БД кому какой URL подписи проставлен**  
   Из корня backend запустите:
   ```bash
   cd dogovor/backend
   .venv/bin/python scripts/verify_signature_mapping.py
   ```
   Скрипт выведет список: ФИО → ссылка на картинку подписи. Откройте каждую ссылку в браузере и убедитесь, что на картинке подпись **этого** человека (Карасева — подпись Карасевой, Ивченко — Ивченко и т.д.).

2. **Проверить файлы на компе**  
   В папке `signatures_to_upload/` имя файла = фамилия того, чья подпись **внутри** файла. Откройте, например, `Карасева.png` — там должна быть подпись Карасевой, а не Ивченко. Если файл перепутан (в `Карасева.png` лежит скан подписи Ивченко), замените содержимое файла или переименуйте файлы правильно, затем заново запустите загрузку (команда выше в разделе «Загрузка в Supabase»).

После исправления файлов обязательно снова выполните загрузку, чтобы в Storage и в БД обновились правильные картинки.

## Где в коде задаётся сопоставление «фамилия ↔ подпись»

1. **Файлы на компе** — в папке `signatures_to_upload/` имя файла = фамилия того, чья подпись **на картинке** (например в `Карасева.png` должна быть подпись Карасевой). Варианты с «ё» в имени файла (Карасёва.png) обрабатываются автоматически.
2. **Код** — список фамилий и имена файлов в Storage задаются в **`dogovor/backend/app/services/signature_upload.py`**:
   - **`STAFF_SURNAMES`** — фамилии в том же порядке, что и в БД (по ним ищется файл `{фамилия}.png` и запись в таблице по `fio`).
   - **`STAFF_STORAGE_NAMES`** — латинские имена файлов в Storage (Kartasheva.png, Kovalenko.png и т.д.), порядок должен совпадать с `STAFF_SURNAMES`.
   - **`STAFF_FILE_ALT`** — если файл назван с «ё» (Карасёва.png, Карташёва.png), скрипт подхватит его по этому словарю.
3. **БД Supabase** — таблица **`dogovor_staff`**: колонка **`fio`** должна **начинаться с фамилии** из `STAFF_SURNAMES` (например «Коваленко Николай Валентинович», «Карасева Виктория Викторовна»). По этому совпадению скрипт проставляет `signature_image_url` после загрузки файла в Storage.
4. После изменения файлов или списка сотрудников заново запустите скрипт загрузки (команда выше).
