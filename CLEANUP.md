# Сворачивание проекта — что почистить

Краткий чек-лист, чтобы ничего не забыть.

---

## 1. Локально (на Mac)

- **Удалить папку проекта** (если не нужна вообще):
  - Либо всю папку: `Desktop/cursor` (в ней и `web/`, и все .md, и `.cursor/rules/`).
  - Либо только чувствительное: удали **`web/.env.local`** (там Supabase URL и ключи). Остальное можно оставить как архив или удалить потом.

- **Никаких глобальных установок** мы не делали (только Node/npm для этого проекта). Удаление папки проекта не трогает систему.

---

## 2. Supabase

- Заходишь в [Supabase Dashboard](https://supabase.com/dashboard) → свой проект.
- **Вариант А — удалить проект:** Settings → General → внизу "Delete project". Освободит квоту и данные.
- **Вариант Б — оставить проект, убрать данные:** в SQL Editor выполни:
  ```sql
  drop table if exists public.collection_items;
  -- если создавал вручную: drop table if exists public.profiles; drop table if exists public.nft_metadata; drop table if exists public.transaction_log;
  ```
  В Storage удали bucket **glasses** (если создавал), чтобы не занимал место.

---

## 3. GitHub

- Если создавал репозиторий и пушил этот код: зайди в репозиторий на GitHub → Settings → внизу "Delete this repository" (или "Archive").
- Если не пушил — локальная папка при удалении сама по себе репо не затронет.

---

## 4. Что ещё не подключали

- **Thirdweb** — не подключали, чистить нечего.
- **Vercel** — не подключали, чистить нечего.
- **Cursor** — правила лежали внутри `.cursor/rules/` проекта; при удалении папки проекта они пропадут.

---

## Итого

| Где           | Действие |
|---------------|----------|
| Локально      | Удалить папку `cursor` на Desktop и/или обязательно удалить `web/.env.local` |
| Supabase      | Удалить проект или только таблицы + bucket `glasses` |
| GitHub        | Удалить или архивировать репозиторий, если создавал |
| Остальное     | Ничего не настраивали |

После этого следов проекта не останется. Удачи с другими идеями.
