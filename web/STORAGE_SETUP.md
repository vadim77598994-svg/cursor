# Шаг 2: Storage для фото очков

Чтобы загружать фото через админку и потом при желании **заменять** их или **добавлять** новые предметы.

---

## 1. Подробно: что сделать с bucket

Bucket в Supabase — это как «папка» в облаке: в неё складываются файлы (твои фото очков). Имя bucket должно быть **ровно `glasses`** — так настроен код админки.

### Шаг 1.1 — Открыть Storage

1. Зайди на [supabase.com/dashboard](https://supabase.com/dashboard) и войди в аккаунт.
2. Выбери **свой проект** (тот, к которому уже подключён фронт по `.env.local`).
3. В левом меню нажми **Storage** (иконка папки/корзины). Откроется раздел хранилища.

### Шаг 1.2 — Создать новый bucket

1. Нажми кнопку **«New bucket»** (или «Create a new bucket»).
2. Появится форма создания bucket.

### Шаг 1.3 — Заполнить форму

- **Name** (имя): введи **`glasses`** — только латиница, без пробелов. Другое имя использовать нельзя: в коде указано именно `glasses`.
- **Public bucket**: включи переключатель **«Public bucket»** в положение «вкл».
  - Зачем: тогда у каждой загруженной картинки будет публичная ссылка вида  
    `https://твой-проект.supabase.co/storage/v1/object/public/glasses/имя-файла.jpg`  
    и галерея на сайте сможет показывать фото без авторизации.
- Остальные поля (File size limit, Allowed MIME types) можно не трогать — по умолчанию подойдёт.

### Шаг 1.4 — Сохранить

1. Нажми **«Create bucket»** (или «Save»).
2. В списке bucket’ов должен появиться **glasses**. Можно зайти в него — пока он пустой, после первой загрузки через админку там появятся файлы.

### Итог по bucket

- Bucket создан с именем **`glasses`**.
- Он **Public** — картинки доступны по прямой ссылке.
- Вручную заливать файлы в Supabase не обязательно: админка **/admin** сама загружает фото в этот bucket при добавлении и редактировании предметов.

**Если картинки на сайте не открываются:** зайди в Storage → bucket `glasses` → справа от имени bucket проверь, что стоит «Public». Если нет — открой настройки bucket и включи «Public bucket».

## 2. Ключ для админки (загрузка файлов)

1. В том же проекте: **Settings** → **API**.
2. Скопируй **service_role** key (secret, не путай с anon).
3. В папке `web/` открой `.env.local` и добавь строку:
   ```env
   SUPABASE_SERVICE_ROLE_KEY=твой_service_role_ключ
   ```
4. Перезапусти `npm run dev`.

## 3. Админка

- Открой **http://localhost:3000/admin**.
- **Добавить:** загрузи фото (файл) или вставь URL картинки, заполни название и атрибуты → «Добавить».
- **Заменить фото:** зайди в «Редактировать» у нужного предмета → выбери новый файл или вставь новый URL → «Сохранить».
- **Увеличить количество:** просто добавляй новые записи через форму «Добавить предмет».

Файлы хранятся в bucket `glasses`; каждая запись в каталоге ссылается на свой URL. Заменить фото = загрузить новый файл и сохранить (URL в записи обновится).
